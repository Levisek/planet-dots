# V4.3 Realismus orbit a tvarů — Design Spec

**Datum:** 2026-05-05
**Verze:** V4.3 (realismus orbit, tvarů, asteroid teaser, tempo slider)
**Předchozí:** V4.2 (3D solar system, Pochopení/Fyzikální mode, formation Beat 1+2)
**Následník (out-of-scope):** V4.4 (formation Beat 3–6, time scrubber + ephemeris, pixel UI polish)

---

## 1. Cíl

Po V4.2 stojí dvě hlavní bolesti:

1. **Pokulhávající fyzika.** Planety obíhají po dokonalých kružnicích. Žádný inclination ani u planet, ani u měsíců — všechno v rovině XZ. Ekvivalentní s pohádkovou ilustrací, ne s reálným systémem.
2. **Měsíce nemají reálné tvary ani plnou texture coverage.** Phobos je koule. Hyperion je koule. Voyagerem fotografované hemisféry Uranových a Neptunových moonů zůstávají na pohled „rozbité".
3. **Pochopení mód jede moc rychle a měsíce jsou moc blízko planetám.**
4. **Retrograde** je implementován jen jako Tritonův oddělený případ.
5. **Mrtvý prostor** mezi Marsem a Jupiterem.

V4.3 řeší všechno najednou jako koherentní balík **„realismus orbit a tvarů"**:

- Eccentricity pro planety + inclination pro vše + sjednocení retrograde mechaniky
- Nové měsíce: Hyperion (chaotic rotation), Phoebe, Sinope, Pasiphae
- Non-uniform tvary nepravidelných moonů + Hyperion vertex displacement + chaotic tumbling
- Texture completion: hybrid kaskáda (real → komunitní → procedural mirror+blur)
- Tempo slider 0.1×–5× v dolním HUD + simulační datum display
- Asteroid teaser: Ceres + Vesta + Pallas + 300-particle ring (struktura kompatibilní s budoucím rozšířením)
- Moon orbit lines v detail view planety

## 3. Filosofie módů

### Pochopení (default)

„Realita projasněná, aby ji lidský oko pochytalo." Idealizace pro pochopení, ne pro výpočet. Konkrétně:

| Veličina | Pochopení | Fyzikální |
|---|---|---|
| Eccentricity (e) | clamp 0.08 | real |
| Inclination — planety (`category: 'planet'`) | clamp 5° | real |
| Inclination — regular moony (`category: 'moon'`) | clamp 15° | real |
| Inclination — irregular tělesa (`category: 'irregular'`, `'dwarf'`) | clamp 30° | real |
| Inclination — suplementární logika | aplikuje se na effective angle (180−inc pokud inc>90°) | real |
| Vzdálenosti planet (semi-major) | compressed sqrt-mapping | linear AU × 3846 |
| Vzdálenosti moonů | compressed | real km přes Keplerovo `T=a^1.5` |
| Globální tempo (`timeScale`) | default 0.5× | default 1.0× |
| Slider rozsah | 0.1× – 5× (oba módy) | 0.1× – 5× |

### Fyzikální

Plné real values. Žádný clamp. Mercury e=0.206, Triton inc=157°, Phoebe inc=175°.

### Per-category clamp jako koncept

`category` field rozhoduje o clamp třídě. Žádný per-objekt override. Tělesa mají genezní rozdíly:

- `planet` — vznikla v ekliptice z protoplanetárního disku, nízký inc (max Mercury 7°)
- `moon` — vznikla s parent planetou nebo zachycená brzy, většinou regulární
- `irregular` — zachycené KBO/asteroidy/kometárního původu, dramatické orbity (Triton, Phoebe, Pasiphae group, Nereid s e=0.75)
- `dwarf` — Ceres a Vesta (Vesta technicky proto-planet, ale velikostí v dwarf bracketu pro účely této vizualizace)

### Suplementární clamp logika

```js
function getInclination(body) {
  const real = body.inclinationDeg;
  if (isFyzikalni()) return real;

  const cap = INCLINATION_CAPS[body.category]; // 5 / 15 / 30
  const effective = real > 90 ? 180 - real : real;
  const clamped = Math.min(effective, cap);
  return real > 90 ? 180 - clamped : clamped;
}
```

**Důsledek pro current C set:**

| Moon | Real inc | Effective | Pochopení output | Drama? |
|---|---|---|---|---|
| Triton | 157° | 23° | **157°** | ✅ Plný flip (effective < cap) |
| Phoebe | 175° | 5° | **175°** | ✅ Plný flip |
| Sinope | 158° | 22° | **158°** | ✅ Plný flip |
| Pasiphae | 151° | 29° | **151°** | ✅ Plný flip (těsně) |
| Nereid | 5° | 5° | 5° | Prograde, mírný |
| Pallas (asteroid, dwarf cap = 30°) | 35° | 35° | 30° | Mírná redukce |

Wow efekt zachován pro všechna high-inc tělesa. Jen Pallas dostane mírnou redukci 35→30 v Pochopení.

## 4. Architektura

### Sjednocená orbit abstrakce

`orbit.js` se stane **jediným** zdrojem orbit math. `planetOrbits.js` přejde na thin wrapper. Měsíce už `orbit.js` používají, planety se přidají.

```
planets.js / moons.js / asteroids.js  (raw data)
    │
    ▼
simMode.js  (mode-aware getters, clamp logika, timeScale)
    │ getEccentricity, getInclination, getMoonPeriod, getOrbitRadius, getOrbitalPeriod, getTimeScale
    ▼
orbit.js  (Kepler solver, applyInclination)
    │ orbitPosition(t, phase, period, a, e, incDeg) → {x, y, z}
    ▼
main.js / planetOrbits.js  (anchor.position update)
    │
    ▼
THREE.Object3D  (planet/moon/asteroid anchor — children scene nebo planet anchor)
```

### Bodydata kategorizace

`bodyData.js` (existing) sjednotí kategorizaci. Field `category` je single source of truth pro clamp logiku:

```js
// planets.js
{ id: 'mercury', category: 'planet', ... }

// moons.js
{ id: 'luna', parent: 'earth', category: 'moon', ... }
{ id: 'triton', parent: 'neptune', category: 'irregular', ... }
{ id: 'phoebe', parent: 'saturn', category: 'irregular', ... }

// asteroids.js
{ id: 'ceres', category: 'dwarf', ... }
{ id: 'pallas', category: 'irregular', ... }
```

### Dvojitý časový kanál

Globální `timeScale` ovládá *jen* orbity a rotace. Formation intro, sun activity, solar wind používají real-time. Detaily v §5 Tok B.

### Modul layout (po V4.3)

## 5. Data flow

### Tok A: Mode-aware hodnoty

```
bodyData (raw e, eReal, inclinationDeg, period, periodReal)
   │
   ▼ getEccentricity(body), getInclination(body), getMoonPeriod(body)
simMode (current mode, clamp per category)
   │
   ▼ orbitPosition(t, phase, period, a, e, inc)
orbit.js (Kepler + applyInclination)
   │
   ▼ {x, y, z}
main.js / planetOrbits / asteroidUpdate
   │
   ▼ anchor.position.set(...)
THREE.Object3D anchor
```

`simMode` getters jsou volány **per-frame** (lazy). Žádný cache, žádná invalidace. Mode change = okamžitý visuální skok přes přepočet positions.

### Tok B: TimeScale + dvojitý časový kanál

```
timeControls.js (UI slider) ──► simMode.setTimeScale(x)
                                       │
                                       ▼ getTimeScale()
main.js mainloop (per frame):
   const dtReal = (now - prevNow) / 1000;
   const dtSim = dtReal * getTimeScale();
   simElapsed += dtSim;       // monotonic pokud timeScale > 0; reverse pokud < 0
   realElapsed += dtReal;     // vždy monotonic

   updatePlanetOrbits(simElapsed)        ▲
   updateMoonOrbits(simElapsed)          │ ovládá timeScale (lze reverse)
   updateAsteroidOrbits(simElapsed)      │
   updateRotations(simElapsed)           ▼

   formationIntro.tick(realElapsed)      ▲
   sunActivity.tick(realElapsed)         │ vždy real-time
   solarWind.tick(realElapsed)           ▼
```

**Reverse playback** (`timeScale < 0`): orbity jdou pozpátku (Kepler je time-symmetric). Formation/wind/sun nereversují. Hyperion freeze (viz §7 E4). Dokumentováno v UI tooltipu.

### Tok C: Detail view → moon orbit lines

```
User klik na planet ──► picking.js ──► detailView.show(planetId)
                                              │
                                              ▼
detailView.js:
  cameraTween na planet
  moonOrbitLines.showFor(planetId):
    pro každý m kde m.parent === planetId:
      sample = sampleKeplerCurve(64, getEccentricity(m), getInclination(m), m.a × parent.radiusPx)
      line = new THREE.LineLoop(sample geom, material)
      line.parent = planetAnchor (dědí axial tilt)
      line.visible = true
  registerOnce(simMode.onModeChange, () => moonOrbitLines.refreshFor(planetId))

User ESC ──► detailView.hide()
                │
                ▼ moonOrbitLines.hideAll()  (dispose geoms, unregister listener)
```

Lines jsou **per-detail-view session**. Žádný permanent cache. Mode switch v detail view → re-sample.

## 6. Implementace per fáze

### F1 — Foundation (mechanika)

**Soubory:**

- **`src/orbit.js`** — rozšířit `orbitPosition(t, phase, period, a, e, incDeg = 0)`:
  - Spočítat `{x, _, z}` v lokální orbital plane (Y=0, jak doposud)
  - Aplikovat `applyInclination({x, 0, z}, incDeg)`:

    ```js
    function applyInclination(pos, incDeg) {
      const r = (incDeg * Math.PI) / 180;
      const c = Math.cos(r), s = Math.sin(r);
      return { x: pos.x, y: -pos.z * s, z: pos.z * c };
    }
    ```
  - Rotuje orbital plane kolem osy X. Pro inc=0 noop. Pro inc=90° = polar. Pro inc=157° = překlopený (Triton).
  - High-e Kepler stabilizace: `solveKepler(M, e, iterations = e > 0.9 ? 12 : e > 0.6 ? 8 : 5)` s improved initial guess pro `e > 0.9`.

- **`src/planetOrbits.js`** — rewrite `orbitalPosition(planet, elapsed)` jako thin wrapper:

  ```js
  import { orbitPosition } from './orbit.js';
  import { getOrbitRadius, getOrbitalPeriod, getEccentricity, getInclination } from './simMode.js';

  export function orbitalPosition(planet, elapsed) {
    const a = getOrbitRadius(planet);
    if (a === 0) return { x: 0, y: 0, z: 0 };
    const period = getOrbitalPeriod(planet);
    const e = getEccentricity(planet);
    const inc = getInclination(planet);
    return orbitPosition(elapsed, planet.initialPhaseRad, period, a, e, inc);
  }
  ```

- **`src/simMode.js`** — rozšíření:
  - `INCLINATION_CAPS = { planet: 5, moon: 15, irregular: 30, dwarf: 30 }`
  - `ECCENTRICITY_CAP = { planet: 0.08, moon: 0.08, irregular: 0.08, dwarf: 0.08 }`
  - `getInclination(body)` — suplementární clamp logika (viz §3)
  - `getEccentricity(body)` — sjednocení existing `getMoonE` na universal getter (deprecate `getMoonE` jako alias během F1, smaž v F4)
  - `_timeScale = 0.5` (init), s onChange při setMode (Pochopení 0.5, Fyzikální 1.0 pokud user nemá explicit override)
  - `_userOverrideTimeScale: boolean` — pokud user hnul slider, neresetovat při mode change
  - `getTimeScale()`, `setTimeScale(x)`, `_timeScaleListeners` array
  - `isRetrograde(body) => body.inclinationDeg > 90` derived helper

- **`src/planets.js`** — každá planeta:
  - `category: 'planet'`
  - `e`, `eReal` (e clamped na 0.08, eReal real). Mercury eReal=0.2056, Mars eReal=0.0934. Ostatní eReal < 0.06 takže e=eReal (no clamp).
  - `inclinationDeg` (real values, ekliptická refernace): Mercury 7.0°, Venus 3.4°, Earth 0°, Mars 1.85°, Jupiter 1.31°, Saturn 2.49°, Uranus 0.77°, Neptune 1.77°.

- **`src/moons.js`** — každý moon:
  - `category: 'moon' | 'irregular'`
  - `inclinationDeg` (vůči ekvatoru rodiče, Wikipedia/NASA Fact Sheet hodnoty)
  - `retrograde` field **odstraněn** kde byl (Triton)
  - Triton: `inclinationDeg: 157`, žádný retrograde flag (suplementární clamp zachová wow v Pochopení)

- **`src/main.js`** — `updateMoonOrbits()` použije `getEccentricity(m)`, `getInclination(m)`. **Smazat** `period × -1` retrograde flip:

  ```js
  // PŘED:
  const scaledPeriod = basePeriod * entry.period * (m.retrograde ? -1 : 1);
  // PO:
  const scaledPeriod = basePeriod * entry.period;
  ```

- **`src/main.js`** mainloop — dvojitý časový kanál (viz §5 Tok B). Konstanta `timeScale` čtená per-frame přes `getTimeScale()`.

  ```js
  export function applySimplexDisplacement(geometry, opts) {
    const { type = 'simplex', amplitude = 0.1, seed = 'default' } = opts;
    const noise = makeSeededSimplex(seed);
    const positions = geometry.attributes.position;
    const normals = geometry.attributes.normal;
    for (let i = 0; i < positions.count; i++) {
      const nx = noise(positions.getX(i), positions.getY(i), positions.getZ(i));
      const offset = nx * amplitude;
      positions.setX(i, positions.getX(i) + normals.getX(i) * offset);
      positions.setY(i, positions.getY(i) + normals.getY(i) * offset);
      positions.setZ(i, positions.getZ(i) + normals.getZ(i) * offset);
    }
    positions.needsUpdate = true;
    geometry.computeVertexNormals();
  }
  ```

  ```js
  const state = {
    omega: [0.13, 0.21, 0.07],
    theta: [0, 0, 0],
    seed: 'hyperion',
    lastSimT: 0,
  };

  export function tickHyperion(simElapsed, mesh) {
    const dt = simElapsed - state.lastSimT;
    if (dt < 0) return;  // reverse playback freeze
    state.lastSimT = simElapsed;

    for (let i = 0; i < 3; i++) {
      const modulation = simplex2(simElapsed * 0.3, i, state.seed) * 0.05;
      state.omega[i] += modulation * dt;
      state.theta[i] += state.omega[i] * dt;
    }
    mesh.rotation.set(...state.theta);
  }
  ```

  ```js
  const J2000_EPOCH = new Date('2000-01-01T12:00:00Z').getTime();
  const EARTH_SIM_PERIOD_SEC = 10;       // Earth perioda v sim time
  const DAYS_PER_SIM_SECOND = 365.25 / EARTH_SIM_PERIOD_SEC;

  export function getSimulationDate(simElapsed) {
    const simulatedDays = simElapsed * DAYS_PER_SIM_SECOND;
    const ms = simulatedDays * 86400 * 1000;
    return new Date(J2000_EPOCH + ms);
  }

  export function formatRelative(simElapsed) {
    const days = Math.floor(simElapsed * DAYS_PER_SIM_SECOND);
    const hours = Math.floor((simElapsed * DAYS_PER_SIM_SECOND - days) * 24);
    const sign = simElapsed < 0 ? '−' : '+';
    return `J2000 ${sign} ${Math.abs(days)}d ${Math.abs(hours)}h`;
  }
  ```

## 7. Edge cases & error handling

### E1: Texture completion fallback chain

```
Pro každý problematický moon (např. miranda):
  try primary URL (komunitní completed map)
    ↓ success → save, done
    ↓ fail → continue
  try fallback URL (Wikimedia Voyager)
    ↓ audit detects >40% dark → run procedural completion
      ↓ success → save with audit flag
      ↓ procedural fails → save raw + flag in textures/audit.json
    ↓ audit clean → save, done
  všechno selhalo → save uniform-grey fallback + log warning
```

Procedural completion: `mirrorAndBlur(img)`:
1. Detect dark region mask (RGB sum < 120 threshold)
2. Mirror flip horizontal
3. Gaussian blur 20px na mirror
4. Alpha blend 0.6 do dark regionu
5. Output JPG

Edge case: primary URL vrátí HTML 404. Existing `fetchBuffer` detekuje (`HTML response (404)`). Beze změny.

### E2: Kepler solver konvergence

Pro vysoké eccentricity (Nereid e=0.75, future Halley e=0.97):
- `e ≥ 0.6` → 8 iterací (default 5)
- `e ≥ 0.9` → 12 iterací + improved initial guess `E = M + e * sin(M)`

Implementace: `solveKepler(M, e, iterations = e > 0.9 ? 12 : e > 0.6 ? 8 : 5)`. Cena: zanedbatelná pro current set (jen Nereid > 0.6).

### E3: Inclination + retrograde sjednocení

`retrograde` flag a `period × -1` mechanika **zrušeny**. High-inc tělesa získají flip přes plane rotation:

| Body | inclinationDeg (Fyzikální) | Pochopení (cap 30 + suplementární) | Mechanika |
|---|---|---|---|
| Triton | 157.0° | 157.0° (effective 23.0 < 30, no clamp) | applyInclination(157.0°) |
| Phoebe | 175.3° | 175.3° (effective 4.7 < 30) | applyInclination(175.3°) |
| Sinope | 158.1° | 158.1° (effective 21.9 < 30) | applyInclination(158.1°) |
| Pasiphae | 151.4° | 151.4° (effective 28.6 < 30) | applyInclination(151.4°) |
| Pallas | 34.84° | 30° (clamped z 34.84) | applyInclination(30°) |
| Nereid | 5.0° | 5.0° | applyInclination(5.0°) |

`isRetrograde(body) => body.inclinationDeg > 90` derived helper pro UI tooltip / info panel.

### E4: Hyperion chaos determinism

Stejný `simElapsed` → stejný `state.theta`. Reverse playback (`dt < 0`): freeze (state nepokračuje). Drobný kompromis dokumentovaný v UI tooltipu („Reverse playback ovládá jen orbity").

### E5: Asteroid particle pick

300 particles v `THREE.Points` jsou **nereagují na pick**. Jen Ceres/Vesta/Pallas jsou klikatelné meshes. Explicit design — 300 named asteroids = V4.5+.

### E6: Saturn ring tilt

V F3 verifikace:
1. Spustit aplikaci, klik na Saturn → detail view
2. Visualně check ring v rovině Saturn ekvátoru (sklon 26.73° vůči ekliptice)
3. Pokud ne → check `saturnRing.js` že ring mesh je `add()` na `saturnAnchor`. Jeden řádek fix.
4. Pochopení mode test: Saturn cap=5° → ring sklon = 5° + 26.73° axial tilt = ?? — ring **dědí** Saturn axial tilt (parent rotation), takže vidíme stále 26.73° v relativní rovině. Inclinace orbital plane Saturn vůči ekliptice je separátní jev.

### E7: User override timeScale

Při mode change (`Pochopení ↔ Fyzikální`):
- Pokud `_userOverrideTimeScale === false` → reset na default per mode (0.5 / 1.0)
- Pokud `true` → zachová user value
- Reset: keystroke `0` → `_userOverrideTimeScale = false`, set default per current mode

### E8: Reverse playback dokumentace

UI tooltip nad reverse button:
> „Reverse playback ovládá jen orbity. Slunce, formace systému a sluneční vítr jdou vždy dopředu."
